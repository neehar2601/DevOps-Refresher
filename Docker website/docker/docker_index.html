What is Docker?

Docker is a platform for building, packaging, and running applications as containers. A container bundles your app and its dependencies into an isolated, reproducible environment that runs identically everywhere.

Mental model: Image = blueprint (immutable snapshot) · Container = running instance (mutable runtime layer on top)

Why containers?

Consistent runtime across laptops, CI, and production. Quick startup. Reproducible builds. No "works on my machine" excuses.

Consistency Speed Portability

Images, containers, registries, Dockerfile, build context, networks, and volumes — the seven building blocks.
